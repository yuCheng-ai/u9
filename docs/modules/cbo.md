# CBO 基础设置 (Common Business Objects)

CBO 是 ERP 系统的基石，负责维护全系统的基础档案、组织架构及跨模块共享的主数据。

## 1. 菜单结构与功能点

### 1.1 组织建模
- **组织机构**：定义法人、组织职能、上下级关系。
- **工作中心**：定义生产制造的基本单元，关联资源与产能。
- **存储地点 (Warehouse)**：维护仓库、货位信息，支持多层级管理。

### 1.2 料品管理 (Item)
- **料品档案**：
    - 基本属性：编码、名称、规格、计量单位、分类。
    - 业务属性：采购属性、销售属性、库存属性、生产属性。
    - 成本属性：计价方法（移动平均、标准成本等）、成本域分配。
- **料品分类**：支持多维分类体系，用于报表统计与权限控制。
- **单位换算**：支持全局单位、料品专属单位换算逻辑。

### 1.3 合作伙伴 (Partner)
- **客户档案**：
    - 多组织分配：一个客户可在多个销售组织下生效，维护不同的信用额度。
    - 收款条件：定义账期、折扣逻辑。
- **供应商档案**：
    - 资质管理：准入审核、证书有效期监控。
    - 付款条件：定义结算方式、付款周期。

### 1.4 基础档案
- **币种与汇率**：维护全球币种及每日汇率。
- **收付款条件**：灵活定义 `Net 30`, `COD` 等复杂条款。
- **税务设置**：定义税种、税率及价税合计计算逻辑。

## 2. 核心业务逻辑

### 2.1 料品多版本逻辑
- 系统支持料品版本管理。当设计变更不影响互换性时，使用小版本；影响互换性时，升级编码。
- **逻辑点**：在订单与生产环节，可指定特定版本或默认使用最新版本。

### 2.2 组织分配逻辑 (Data Allocation)
- **逻辑点**：主数据（如料品）由“创建组织”创建后，必须经过“分配”流程才能在“业务组织”中使用。这确保了集团管控的一致性。

### 2.3 自动编码规则
- 支持自定义编码逻辑（前缀 + 日期 + 流水号），支持不同组织使用不同编码规则。
