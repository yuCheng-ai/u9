# 多账簿核算体系 (Multi-Book)

## 概述
随着企业全球化和上市需求的增加，单一的会计准则往往难以满足监管要求。U9 cloud 的“多账簿”架构允许企业针对同一笔业务，按照不同的会计准则（如 CAS、IFRS、US GAAP）进行并行核算，实现“一次业务录入，多套账簿产出”。

## 1. 核心架构：法人、核算组织与账簿
- **法人实体**: 法律主体。
- **核算组织**: 负责成本归集和凭证生成的单元。
- **账簿 (Ledger)**: 具体的核算载体。一个核算组织可以关联多个账簿。

## 2. 实时并账逻辑 (Real-time Dual-Accounting)
这是 U9 的核心优势。当业务单据（如出货单）发生并抛送到“自动会计平台”时：
- **逻辑点**: 系统会根据预设的映射规则，同时在“主账簿”（如 CAS）和“调整账簿”（如 IFRS）生成会计凭证。
- **差异处理**: 对于准则间的差异（如折旧年限不同、收入确认时点不同），系统支持通过“调整凭证”仅在特定账簿中记录差异，而不影响其他账簿。

## 3. 多账簿的应用场景

### 3.1 跨国集团核算
- **主账簿**: 本地准则（用于税务申报）。
- **汇报账簿**: 集团总部准则（用于集团合并）。
- **折算账簿**: 自动进行外币折算（Translation），按期末汇率生成汇报币种账簿。

### 3.2 管理会计与阿米巴核算
- **管理账簿**: 按照内部考核规则（如内部交易价、特定的分摊逻辑）进行核算，用于评价阿米巴的盈利能力。
- **财务账簿**: 严格遵循外部审计准则。

## 4. 自动会计平台 (AEP) 的支持
- **映射规则**: 可以配置同一业务在不同账簿中映射到不同的会计科目。
- **异步处理**: 业务人员只需关注单据录入，财务端的“多账簿凭证”由 AEP 后台自动、异步生成。

## 5. 价值总结
- **合规性**: 轻松应对跨国审计和上市监管。
- **效率**: 消除人工调整账项的繁重工作。
- **透明度**: 随时比对不同准则下的利润差异，为决策提供多维财务视图。
