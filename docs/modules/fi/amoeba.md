# 阿米巴经营会计 (Amoeba)

## 概述
U9 阿米巴经营会计是基于稻盛和夫经营哲学的落地实践。它通过将企业划分为小的经营单元（阿米巴），建立“人人都是经营者”的机制。系统支持多维核算体系，帮助企业实现经营数据的实时化、透明化，从传统的事后财务核算转向事前的经营模拟与事中的精细控制。

## 业务价值
- **量化经营贡献**: 准确评估每个最小核算单元的盈利能力，消除经营死角。
- **培养经营人才**: 让员工像老板一样关注损益，建立结果导向的激励机制。
- **快速响应市场**: 通过日核算机制，管理者可在第一时间发现业务异常并调整决策。
- **业财深度融合**: 将复杂的生产经营活动转化为直观的损益数据，实现语言统一。

## 核心概念
- **核算单元**: 最小的经营实体，可以是车间、班组、职能部门或特定项目组。
- **内部转移价格**: 单元间协作的定价机制。U9 支持多种定价策略，是核算盈亏的核心杠杆。
- **单位时间核算**: 核心指标（总附加值 / 总劳动时间），衡量单位效率。

## 核心功能模块

### 1. 灵活的组织架构 (Dual-Track System)
- **逻辑点**: 支持“行政架构”与“核算架构”并行。阿米巴单元可以跨法人、跨部门组建，不影响法定财务汇报。
- **动态调整**: 随着业务变化，阿米巴单元可快速合并、拆分，历史数据可追溯。

### 2. 多维度内部定价策略
- **市场对标价**: 参考外部市场价格进行内部结算。
- **成本加成价**: 在标准成本或实际成本基础上增加固定利润比例。
- **协商定价**: 允许各单元在系统预设范围内进行议价。

### 3. 自动化的核算引擎
- **业务驱动**: 生产领料、产成品入库、销售发货等业务行为实时触发内部结算。
- **费用分摊规则**:
    - **直接计入**: 能够明确归属的费用直接进入单元损益。
    - **分摊计入**: 公共费用（如房租、职能支出）按面积、人数、产值等自定义权重自动分摊。

### 4. 经营报告与看板
- **阿米巴损益表**: 实时生成各单元的收入、支出、利润数据。
- **单位时间核算表**: 重点监控人均效能与时间产出比。
- **多级穿透查询**: 从汇总数据一键穿透至原始业务单据（如领料单、发票）。

## 核心业务场景

### 场景 A：制造环节的内部结算
- **流程**: 第一工序（阿米巴 A）完成后，半成品流转至第二工序（阿米巴 B）。
- **逻辑**: 系统自动按“内部协议价”扣减 B 的预算，增加 A 的收入。

### 场景 B：公共职能部门的“服务化”
- **流程**: 行政部或 IT 部作为“支撑型阿米巴”。
- **逻辑**: 根据提供的服务次数或支持时长，向业务部门收取“服务费”，实现职能部门从费用中心向利润中心的转化。

## 技术架构支持
阿米巴核算的底层依托于 U9 的核心技术平台：
1. **[多账簿体系](file:///Users/dingshaoxiong/yuCheng-ai/u9/docs/modules/fi/multi-book.md)**: 通过建立独立的“管理账簿”，实现阿米巴核算与法定财务核算的解耦与并存。
2. **自动会计平台 (AEP)**: 业务单据通过 AEP 的“双重映射”规则，同步生成财务凭证与阿米巴经营记录。
3. **大数据处理引擎**: 支持海量业务单据的实时结算，确保“日清日结”的性能。

## 落地建议 (PM Insights)
- **划小核算单元要适度**: 过细会导致管理成本过高，建议先按车间/大部门划分，成熟后再下沉。
- **内部定价是关键**: 定价需公开透明，建议引入“仲裁委员会”机制处理单元间的定价纠纷。
- **从看板开始**: 实施初期可以先做“损益看板”，让员工建立成本意识，再逐步推行考核激励。

## U9 核心竞争力
1. **多维度核算**: 同时支持法人核算、管理核算与阿米巴核算，三账合一。
2. **实时性**: 依托 U9 业财一体化底座，数据无延迟，支持“日结算”。
3. **灵活性**: 支持复杂的组织架构变动，满足快速成长的企业需求。
