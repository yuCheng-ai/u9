# 销售订单执行 (Sales Process)

## 1. 概述
销售订单执行模块是企业收入增长的核心引擎。它不仅负责记录销售契约，更重要的是驱动整个供应链的运作。在 U9 中，销售订单作为“源头需求”，自动触发 ATP（可用量检查）、采购申请、生产订单或库存出库，实现真正的“以销定产/定采”。

## 2. 关键业务流程详解

### 2.1 销售订单管理 (SO)
- **多类型支持**: 标准销售、出口贸易、零售业务、样机赠送等。
- **价格与折扣引擎**: 实时调用 [销售价格体系](./sales-price) 中的规则，自动匹配最优价格。
- **变更处理**: 严格的订单变更控制，支持在订单执行过程中根据生产进度动态调整。

### 2.2 出货申请与通知 (Shipping Request)
- **职能隔离**: 销售部发起申请，通知仓库/物流部。
- **备料锁定**: 出货申请通过后，系统自动锁定相关库位的实物库存，防止“抢货”现象。
- **拼单逻辑**: 支持将多个订单合并到同一张出货申请单，优化物流配送成本。

### 2.3 退货管理 (RMA)
- **闭环追溯**: 退货单必须关联原出货单或订单，确保单价、数量、批次准确无误。
- **质检入库**: 退回物料须经过检验，根据结果判定入“合格库”、“报废库”或“返工库”。
- **财务退款/折抵**: 自动生成红字应收账单，支持与后续交易对冲。

### 2.4 直运业务 (Drop Shipping)
- **三方协同**: 销售组织接单，由供应商直接发货给客户。系统自动同步生成“采购订单”与“销售出货”，简化跨组织结算。

## 3. 实时监控与预警
- **交付达成率**: 实时看板展示订单从接单到发货的各环节时长。
- **异常预警**: 针对逾期未发货、信用拦截订单自动推送到相关负责人的工作台。
