# LRP 批次需求计划 (Lot Requirements Planning)

## 概述
LRP 是 U9 区别于传统 ERP 的核心竞争力之一。在传统的 MRP 中，库存是“大锅饭”，难以追溯某个零件是为哪个订单采购的。而 **LRP (批次需求计划)** 实现了从销售订单到采购、生产的“全链路硬关联”。

## 1. LRP 的核心理念：专料专用
- **逻辑点**: LRP 以“需求来源单据”（通常是销售订单或预测单）为核心，在整个计划、采购、入库、领料过程中，始终携带该单据的“批次号/项目号”。
- **价值**: 
  - **精准追溯**: 随时查看某个销售订单的原材料到货进度。
  - **成本归集**: 支持按订单计算实际成本。
  - **呆滞控制**: 避免因通用库存导致的物料被挪用。

## 2. 运算模型：Pegging (供需追溯)
LRP 引擎通过 Pegging 链条将供需紧紧锁定：
- **需求端**: 销售订单 A。
- **中间端**: 生产订单 B (关联 A)。
- **供给端**: 采购申请 C (关联 B 和 A)。
- **库存端**: 项目库存 D (仅限 A 使用)。

## 3. 核心功能与操作

### 3.1 齐套分析 (Kitting Analysis)
- **逻辑点**: 在下达生产前，LRP 可以针对特定订单进行齐套检查。
- **结果**: 清晰显示“欠料清单”，并自动计算出最早可以开工的日期。

### 3.2 批次合并与拆分
- **合并**: 虽然要专料专用，但对于小额标准件（如螺丝），支持将多个 LRP 批次的需求合并为一个采购订单，以获取批量折扣。
- **拆分**: 当一个大订单部分提前交付时，支持 LRP 计划的拆分处理。

### 3.3 紧急插单处理
- **场景**: 客户临时增加一个急单。
- **LRP 优势**: LRP 可以在不影响其他订单计划的前提下，单独为该订单运行计划，快速识别资源冲突。

## 4. LRP vs. MRP：如何选择？

| 维度 | MRP (物料需求计划) | LRP (批次需求计划) |
| :--- | :--- | :--- |
| **适用场景** | 稳定的大规模量产、备库生产 | 离散制造、ETO (按订单设计)、急单多 |
| **库存逻辑** | 通用库存，先进先出 | 专料专用，按单锁定 |
| **计算压力** | 全局计算，压力大 | 局部计算，速度快 |
| **追溯深度** | 弱（需通过库存流水追溯） | 强（原生链路锁定） |

## 5. 最佳实践：混合模式
在 U9 中，企业通常采用混合模式：
- **A 类关键件/长周期件**: 采用 LRP 管理，确保供应安全。
- **C 类通用件/标准件**: 采用 MRP 或 ROP (再订货点) 管理，提高库存周转率。
