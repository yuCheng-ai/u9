# 核心业务架构与多组织逻辑

本章节详细描述 ERP 系统的核心底层架构。作为一款竞争 U9 的产品，必须具备强大的多组织管理能力，支持复杂的集团化运营模式。

## 1. 组织模型 (Organization Model)

### 1.1 组织类型
系统支持以下核心组织类型，并允许一个实体具备多种职能：
- **法人据点 (Legal Entity)**：法律意义上的公司，具备独立的税务登记。
- **核算组织 (Accounting Organization)**：负责财务核算的实体，通常关联一套账簿。
- **物流组织 (Inventory Organization)**：负责库存管理、收发货流程。
- **生产组织 (Manufacturing Organization)**：负责生产计划、车间执行、BOM 管理。
- **销售组织 (Sales Organization)**：负责客户合同、销售订单处理。
- **采购组织 (Purchasing Organization)**：负责供应商协议、采购申请与下单。

### 1.2 组织间关系
- **组织树**：支持多套组织树逻辑（如行政树、业务协同树、财务合并树）。
- **多角贸易 (Inter-company Trade)**：当销售组织与物流组织不一致时，系统自动生成内部交易单据。

## 2. 财务架构 (Financial Architecture)

### 2.1 多账簿 (Multi-Book)
- **主账簿**：遵循当地会计准则（如中国准则）。
- **报告账簿**：支持多准则（如 IFRS、US GAAP）转换，实现实时合并。
- **本币与汇率**：支持组织级别的本位币设置及全局/组织级汇率体系。

### 2.2 成本域 (Cost Domain)
- 定义：成本核算的边界。一个成本域可以包含多个物流组织。
- 逻辑：同一成本域内的料品移动不产生毛利，跨成本域移动视为内部销售。

## 3. 核心业务逻辑

### 3.1 实时企业理念
- **单据驱动**：业务单据（如收货单）实时驱动下游单据（如应付单）及会计分录。
- **自动会计平台**：定义业务事件到会计分录的映射规则，实现“业务即财务”。

### 3.2 共享服务模式
- **主数据共享**：料品、客户、供应商在集团层面统一维护，各组织申请分配使用。
- **集中采购/集中支付**：支持集团代买代付逻辑，自动处理内部往来。
