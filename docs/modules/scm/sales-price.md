# 销售价格体系 (Sales Price)

## 概述
在复杂的 B2B 贸易中，价格从来不是单一的数字。U9 级的销售价格体系支持多币种、多层级、带有效期的价格策略，能够满足集团企业针对不同客户、不同地区、不同销量的差异化定价需求。

## 1. 价格表 (Price List)
价格表是价格存储的载体，支持以下维度：
- **价表类型**: 
  - **销售价表**: 针对客户的最终售价。
  - **协议价表**: 长期合同约定的价格。
- **控制属性**: 
  - **币种与汇率**: 支持按原币定价，下单时根据实时汇率自动换算。
  - **含税标识**: 明确价格是否包含增值税。
  - **有效期**: 通过“生效日期”和“失效日期”管理价格版本。

## 2. 取价策略 (Pricing Strategy)
当用户录入销售订单时，系统会根据预设的“取价方案”自动检索价格，避免人工录入错误。

### 2.1 取价优先级
常见的检索顺序（可配置）：
1. **特价协议**: 针对特定客户、特定期间的专项价格。
2. **客户价表**: 专门为该客户定制的价格清单。
3. **客户等级价表**: 按“金牌客户”、“银牌客户”等分类定价。
4. **标准价表**: 适用于所有客户的公开零售价。

### 2.2 批量价格 (Volume Pricing)
支持阶梯定价逻辑：
- 数量 1-100: 单价 10.00
- 数量 101-500: 单价 9.50
- 数量 > 500: 单价 9.00

## 3. 折扣与加价 (Discounts & Surcharges)
在基础单价之上，系统还支持多重折扣运算：
- **行折扣**: 针对订单中某一行的物料进行折扣。
- **整单折扣**: 针对整笔业务的返点。
- **加价规则**: 如针对小额订单收取的“包装费”或“紧急处理费”。

## 4. 价格控制与审批
- **最低限价控制**: 严禁低于成本价或预设最低售价下单，否则系统报错或触发高管审批流。
- **价格版本追溯**: 记录价格修改的每一个版本，可随时查询历史某天的执行价格。

## 5. 跨组织取价
在多组织协同场景下，销售组织可以取生产组织的结算价作为参考，并在此基础上加上固定比例的毛利，自动生成对外售价。
