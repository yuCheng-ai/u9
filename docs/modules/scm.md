# SCM 供应链管理 (Supply Chain Management)

SCM 模块覆盖了企业从采购、销售到库存管理的全过程，支持多组织协同与复杂的物流场景。

## 1. 菜单结构与功能点

### 1.1 销售管理 (SO)
- **销售预测**：基于历史数据与市场预估，驱动采购与生产计划。
- **销售订单**：
    - 价格协议：支持复杂的阶梯价格、促销折扣及多币种定价。
    - 信用控制：实时检查客户信用额度，支持超额冻结。
    - 可承诺量 (ATP) 检查：录单时实时计算库存及预计产出是否满足交期。
- **出货计划**：安排物流运输，生成出货通知。

### 1.2 采购管理 (PO)
- **采购请购**：内部部门发起需求申请。
- **采购订单**：
    - 委外采购：支持原材料外发、加工费结算的委外模式。
    - 协议采购：基于年度框架协议进行多次下单。
- **收货单**：记录供应商到货情况，触发质检流程。

### 1.3 库存管理 (INV)
- **入库/出库管理**：包括杂收、杂发、调拨单。
- **形态转换**：处理料品拆解、组装及规格变更。
- **盘点管理**：支持周期盘点 (Cycle Count) 与年终大盘。
- **批次/序列号管理**：实现全生命周期追溯。

### 1.4 质量管理 (QC)
- **来料检验 (IQC)**：关联采购收货，判定合格/不合格。
- **过程检验 (IPQC)**：关联生产工序汇报。
- **出库检验 (OQC)**：销售发货前的最终品质确认。

## 2. 核心业务逻辑

### 2.1 多角贸易逻辑 (Inter-company)
- **场景**：A 组织接单，B 组织生产并直接发货给客户。
- **逻辑点**：系统自动在 A、B 组织间生成内部采购订单与内部销售订单，并根据内部结算价计算往来账，实现“物流直达，财务结算”。

### 2.2 跨组织调拨 (Transfer)
- **一步调拨**：库存实时从 A 转到 B（适用于同园区）。
- **两步调拨**：A 发货 -> 在途仓 -> B 收货（适用于异地运输，支持在途库存监控）。

### 2.3 需求追溯逻辑 (Pegging)
- **逻辑点**：从销售订单出发，可以穿透查看关联的生产订单、采购订单及当前进度，确保交付透明。

### 2.4 批次属性继承
- **逻辑点**：生产领料时，原材料的批次属性（如产地、等级）可自动继承到产成品批次中。
